##### Функциональные требования для реализации курсовой работы на тему "Разработка мультиплеерной игры "Тетрис" с использованием JavaScript (React)"

## Основные модули системы

<center><h3> === Лобби и настройка игры === </h3></center>

1. Система должна предоставлять выбор количества игроков (1-3)
2. Система должна позволять игрокам вводить свои имена
3. Система должна отображать список готовых игроков
4. Система должна запускать игру когда все игроки готовы

<center><h3> === Игровая механика Тетриса === </h3></center>

1. Система должна генерировать 7 стандартных фигур (I, O, T, S, Z, J, L)
2. Система должна управлять фигурами через клавиатуру (стрелки, пробел для поворота)
3. Система должна реализовать гравитацию с ускорением по уровням
4. Система должна проверять коллизии с границами поля и упавшими блоками
5. Система должна очищать заполненные линии и начислять очки
6. Система должна увеличивать уровень каждые 10 очищенных линий

<center><h3> === Система атак и защиты === </h3></center>

1. Система должна генерировать мусорные линии при одновременной очистке нескольких линий:
   - 2 линии → 1 линия мусора
   - 3 линии → 2 линии мусора
   - 4 линии → 4 линии мусора
2. Система должна добавлять атаки в очередь целевого игрока
3. Система должна позволять гасить входящие атаки собственными очистками линий
4. Система должна поднимать мусор снизу поля если очередь не погашена
5. Система должна позволять выбирать цель атаки при игре втроем

<center><h3> === Мультиплеер === </h3></center>

1. Система должна отображать поля всех игроков одновременно в одной вкладке
2. Система должна обрабатывать управление от нескольких игроков с разными наборами клавиш
3. Система должна показывать очередь атак для каждого игрока
4. Система должна определять проигравших (переполнение поля) и продолжать игру для оставшихся
5. Система должна объявлять победителя когда остается один игрок

<center><h3> === Результаты и статистика === </h3></center>

1. Система должна отображать итоговую таблицу результатов
2. Система должна показывать для каждого игрока: место, очки, линии, уровень
3. Система должна предоставлять кнопку "Играть снова"

## Use Case №1: игра втроем с атаками
### Участники
Три игрока (Player1, Player2, Player3)
### Предусловия
- Игроки собрались за одним компьютером
### Основной сценарий
1. Игроки выбирают "3 игрока" в лобби
2. Игроки вводят имена для каждого участника
3. Нажимается кнопка "Начать игру"
4. Система запускает игру и отображает 3 игровых поля
5. Каждый игрок управляет своими фигурами независимо
6. Player1 очищает 4 линии одновременно (Tetris)
7. Система генерирует 4 линии мусора для атаки
8. Player1 выбирает Player3 как цель атаки
9. Система добавляет 4 линии в очередь атак Player3
10. Player3 видит индикатор входящей атаки
11. Player3 очищает 2 линии (Double)
12. Система гасит 1 линию мусора из очереди Player3
13. Система добавляет оставшиеся 3 линии мусора снизу поля Player3
14. Поле Player2 переполняется (игра окончена для Player2)
15. Система помечает Player2 как "Game Over"
16. Игра продолжается между Player1 и Player3
17. Поле Player3 переполняется
18. Система объявляет Player1 победителем
19. Система показывает экран результатов с местами и статистикой
### Результат
Player1 победил, используя атаки на Player3, все игроки видят финальную таблицу результатов


## Use Case №2: одиночная игра с прогрессией
### Участники
Один игрок
### Предусловия
- Игрок находится в лобби
### Основной сценарий
1. Игрок выбирает "1 игрок" в лобби
2. Игрок вводит свое имя
3. Игрок нажимает "Начать игру"
4. Система запускает игру с начальным уровнем 1
5. Система генерирует первую фигуру и показывает следующую в превью
6. Игрок управляет фигурой стрелками (влево, вправо, вниз)
7. Игрок поворачивает фигуру пробелом
8. Фигура падает и фиксируется на дне
9. Игрок очищает 10 линий
10. Система повышает уровень до 2
11. Система увеличивает скорость падения фигур
12. Игрок продолжает игру на новом уровне
13. Поле переполняется
14. Система показывает "Game Over" с финальной статистикой
15. Система отображает: очки, линии, достигнутый уровень
### Результат
Игрок прошел несколько уровней и увидел свою статистику
